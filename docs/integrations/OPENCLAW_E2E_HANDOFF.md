# NeuraMinds OpenClaw E2E Handoff

## Scope
This handoff defines the external partner test contract for OpenClaw against NeuraMinds on Base. It covers:

- direct MCP over HTTP JSON-RPC
- stdio MCP bridge mode
- x402 payment-gated MCP behavior
- XMTP runtime health precondition
- seeded market/agent requirements for realistic test loops

## Canonical Inputs
Use the environment-specific values generated by:

- `npm run openclaw:pack -- --environment production --api-url <api> --web-url <web>`
- `npm run openclaw:pack -- --environment staging --api-url <api> --web-url <web>`

Artifacts are written to:

- `docs/integrations/openclaw-e2e-pack-production.json`
- `docs/integrations/openclaw-e2e-pack-staging.json`

## Required Headers
For HTTP MCP calls:

- `content-type: application/json`
- `accept: application/json`
- `x-client-id: <stable partner client id>`

For paid requests:

- `x-payment: resource=<...>;amount_microusdc=<...>;nonce=<...>;expires_at=<...>;tx_hash=<...>;signature=<...>`

## Dual MCP Paths
### 1) Direct HTTP JSON-RPC
Profile file:

- `config/openclaw/neuraminds-mcp.direct-http.json`

Endpoint:

- `POST /v1/web4/mcp`

### 2) Stdio Bridge
Profile file:

- `config/openclaw/neuraminds-mcp.stdio.json`

Command:

- `node scripts/mcp-server.mjs`

Bridge env:

- `NEURAMINDS_API_BASE_URL=<api>/v1`

## Runtime Gates
Before partner execution, all must pass:

- `GET /v1/web4/runtime/health`:
  - `components.mcp.ready = true`
  - `components.x402.ready = true`
  - `components.xmtp.ready = true`
  - `fullWeb4Ready = true`
- `GET /v1/evm/markets?limit=<n>` meets seeded minimum
- `GET /v1/evm/agents?active=true&limit=<n>` meets seeded minimum

## x402 Contract
Quote lifecycle:

1. fetch quote via `GET /v1/payments/x402/quote?resource=<resource>`
2. submit onchain payment to quoted receiver
3. attach proof via `x-payment` (HTTP) or `arguments.payment` (MCP tool call)
4. retry before quote expiry

Expected unpaid behavior (when x402 is enabled):

- MCP `tools/call` for premium routes returns `status=402` with error code `PAYMENT_REQUIRED`

## Error Envelope
MCP/x402 errors should follow:

- `code`
- `reason`
- `retryable`
- `quote` (when payment is required)

## Deterministic Validation Commands
Runtime + dual transport:

- `npm run openclaw:e2e -- --mode both --api-url <api> --require-full-web4 --min-markets 3 --min-agents 1`

Strict launch readiness with OpenClaw gate:

- `bash scripts/launch-readiness.sh --strict --mode=production --api-url=<api> --web-url=<web> --chain-mode=base --require-full-web4 --min-evm-markets=3 --min-evm-agents=1 --run-openclaw-e2e --openclaw-mode=both`

Seed pilot markets/agents:

- `npm run base:seed:pilot`

## Pass/Fail Contract
Pass requires all of:

1. Runtime gate passes with `fullWeb4Ready=true`.
2. Seeded data thresholds pass (markets and agents).
3. Direct MCP path passes (`initialize`, `tools/list`, `tools/call`, `resources/list/read`, `prompts/list/get`).
4. Stdio MCP path passes the same method set.
5. x402 unpaid premium-call behavior is enforced as `PAYMENT_REQUIRED`.

Fail is any required check failure in `openclaw-e2e-*.json` reports.

## Freeze Rule During Partner Window
During partner execution window:

- freeze non-critical deploys
- no breaking API/interface changes
- no secret/address drift
- keep indexer, matcher, payout, XMTP bridge, and MCP processes continuously running
